# NOTE: es305_fw.bin / es305_fw_bypass.bin (Audience ES705 voice processor
# firmware) - CONFIRMED PRESENT in stock vendor/firmware/ despite the
# xcover3velte kernel defconfig having no SND_SOC_ES* codec enabled and no
# matching dts node. Carried over for fidelity to stock (harmless if
# genuinely unused - the driver that would read it isn't built), but this
# is a real discrepancy worth being aware of: either the codec chip is
# physically present but simply not wired up in this kernel config/dts, or
# these blobs are dead weight shared from a common vendor image across
# multiple exynos3475 SKUs. Not re-enabling the kernel config based on this
# alone without further evidence the chip is actually used on this device.
